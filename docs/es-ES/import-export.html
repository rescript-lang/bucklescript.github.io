Import & Export

Exportar

BuckleScript permite compilar a:

  • CommonJS (require('myFile'))
  • Módulos ES6 (import * from 'myFile')

El formato de salida es configurable en bsb, descrito en una sección más adelante.

Por defecto, cada enlace let es exportado. Si sus valores son seguros para usar en el lado JS, puedes directamente requerir el archivo JS generado y usarlos (¡ve el mismísimo archivo JS!).

Para exportar unos pocos let seleccionados, simplemente añade un archivo de interface que esconda algunos de los enlaces let.

Exportar un valor por default ES6

Si tu proyecto JS está usando módulos ES6, es posible que estés exportando & importando algunos valores por defecto:

// student.js
export default name = "Al";

// teacher.js
import studentName from 'student.js';

Técnicamente, ya que un archivo BuckleScript mapea a un módulo, no hay algo como una "exportación" por defecto, sólo las nombradas. Sin embargo, hemos hecho una excepción con el módulo de compatibilidad por defecto cuando haces lo siguiente:

(* FavoriteStudent.ml *)
let default = "Bob"

/* FavoriteStudent.ml */
let default = "Bob"

Ahora puedes requerir el predeterminado como un lado normal JS:

// teacher2.js
import studentName from 'FavoriteStudent.js';

Note: the above JS snippet only works if you're using that ES6 import/export syntax in JS. If you're still using require, you'd need to do:

let studentName = require('FavoriteStudent').default;

Importar

Use bs.module. It's like a bs.val that accepts a string that's the module name or path.

external dirname: string -> string = "dirname" [@@bs.module "path"]
let root = dirname "/User/chenglou"

[@bs.module "path"] external dirname : string => string = "dirname";
let root = dirname("/User/chenglou");

Output:

var Path = require("path");
var root = Path.dirname("/User/chenglou");

Note: the string inside bs.module can be anything: "./src/myJsFile", "@myNpmNamespace/myLib", etc.

Importar un Valor Por Defecto

By omitting the payload to bs.module, you bind to the whole JS module:

external leftPad: string -> int -> string = "./leftPad" [@@bs.module]
let paddedResult = leftPad "hi" 5

[@bs.module] external leftPad : string => int => string = "./leftPad";
let paddedResult = leftPad("hi", 5);

Output:

var LeftPad = require("./leftPad");
var paddedResult = LeftPad("hi", 5);

Importar un Valor ES6 Por Defecto

This is a recurring question, so we'll answer it here. If your JS project is using ES6, you're likely using Babel to compile it to regular JavaScript. Babel's ES6 default export actually exports the default value under the name default. You'd bind to it like this:

external studentName: string = "default" [@@bs.module "./student"]
let _ = Js.log studentName

[@bs.module "./student"] external studentName : string = "default";
Js.log(studentName);

Output:

var Student = require("./student");
console.log(Student.default);
