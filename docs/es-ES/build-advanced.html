Personalizar reglas (Generadores de soporte)

Usted debe utilizar algunos pre-procesadores para generar códigos repetitivos durante el desarrollo.

Nota: pre-procesadores pueden ser clasificados en dos categorías:

  • Dependientes del sistema, que deben ser retrasados hasta correr en las máquinas de los usuarios.
  • Independientes del sistema: lex, yacc, m4, re2c, etc, que pueden ser ejecutados en cualquier momento.

BS ha construido un soporte para la compilación condicional, que satisface el primer punto. Esta sección es para el segundo punto.

Un ejemplo, dónde bsb usa ocamlyacc:

{
  "generadores" : [
    {
      "nombre" : "ocamlyacc",
      "comando": "ocamlyacc $in"
    }
  ],
  "fuentes" : {
    "dir" : "src",
    "generadores" : [
      {
        "nombre" : "ocamlyacc",
        "edge" : ["test.ml", "test.mli", ":", "test.mly"]
      }
    ]
  }
}

ocamlyacc generará los artefactos test.ml y test.mli en el mismo directorio que test.mly.

Nota: Recomendamos que revises en los archivos generados, ya que esto elimina las dependencias como ocamlyacc para los usuarios finales.

Cuando se desarrolla un proyecto, bsb hará un seguimiento de las dependencias entre test.ml y test.mly correctamente. Cuando sea liberado como un paquete, bsb eliminará esta dependencia, para que así los usuarios sólo necesitarán el test.ml generado. Para ayudar en la prueba este comportamiento en el modo de desarrollo, los usuarios pueden establecerlo manualmente:

{
  "cut-generators" : true
}

Esto previene a bsb de regenerar test.ml cada vez que test.mly cambie.
