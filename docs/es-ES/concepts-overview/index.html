Concepts Overview

Antes de empezar las siguientes secciones, aquí hay unas cosas muy útiles a saber:

OCaml

Es el respaldo de BuckleScript. BuckleScript es un fork de OCaml, específicamente v4.02.3 (¡actualización inminente!). Este sitio de documentación asume conocimiento básico de OCaml; puedes aprender de OCaml mediante Real World OCaml o, si estás aprendiendo Reason de igual manera, empieza con el sitio de Reason.

Este sitio de documentación cubrirá mayormente los pedacitos extras que hemos añadido a OCaml para habilitar una buena interoperabilidad JS.

Reason

Una alternativa de sintaxis para OCaml que es más familiar para programadores JS y reduce la sobrecarga de aprendizaje un poco. Reason trabaja con OCaml, así que aparte de la sintaxis comparten muchas cosas más. Esto también significa que Reason trabaja con BuckleScript. El último tiene una compatibilidad de primera clase para algunas otras utilidades de Reason.

OPAM

Es el administrador de paquetes oficial de OCaml. Ya que BuckleScript usa NPM y Yarn, no necesitarás OPAM cuando empiezas. Lo mencionaremos para flujos de trabajo avanzados.

External/Interop/FFI

Todas estas son jergas para trabajar con BuckleScript <-> JavaScript.

External: in the context of BuckleScript, one of the primary ways to use a JS value.

Interop: short for "interoperability".

FFI: Foreign Function Interface. The general term for things like "external", "wrapper" and "interop". Basically means calling a value from the other language. Prefer the first two terms.
