JSON

现在有好几种方式可以使用JSON，但很快应该会把这些方式进行整合和统一

不安全转换

模拟 JavaScript 的 JSON 转换。

Parse

使用特殊的外部标识（最后的手段）

type data = {name: string} [@@bs.deriving abstract]
external parseIntoMyData : string -> data = "parse" [@@bs.scope "JSON"][@@bs.val]

let result = parseIntoMyData "{\"name\": \"Luke\"}"
let n = result |. name

[@bs.deriving abstract]
type data = {name: string};

[@bs.scope "JSON"] [@bs.val]
external parseIntoMyData : string => data = "parse";

let result = parseIntoMyData("{\"name\": \"Luke\"}");
let n = nameGet(result);

输出：

var result = JSON.parse("{\"name\": \"Luke\"}");
var n = result.name;

其中JSON数据可以是您假定的任何类型。 正如您所看到的, 这将直接编译为 JSON. parse 的调用。 与常规 JS 一样, 这是方便的, 但不能保证数据是正确的类型, 甚至在语法上是有效的。

Stringify

因为 JSON.stringify 略微 比 JSON.parse更安全, 所以我们提供了Js. Json。 它会被编译为 JSON. stringify。

正确使用 Js. Json

从技术上讲, 处理 JSON 的正确方法是递归地分析每个字段, 并相应地处理无效数据。 Js. Json 提供了这样的低级构建块。 请参阅 API 文档中的示例。

高级助手

我们有一个社区维护的，伪官方的JSON库， bs-json。 对编译器感兴趣的人也可以check out JsonCodec。 利用上一节的低级别 API，两者都提供了更方便的 JSON 解析/打印帮助程序,。

我们未来即将提供一个更好的, 官方的JSON解决方案。敬请关注!
