Concepts Overview

在开始下几个章节之前，你需要知道以下这些概念

OCaml

Ocaml是BuckleScript的基础 BuckleScript 是Ocaml的一个分支，特别是v4.02.3。 这个文档假设你已经掌握了BuckleScript基础知识，如果你没有BuckleScript知识，你可以通过[这个地址]](https://realworldocaml.org/)学习Ocaml

当前文档只介绍我们额外添加到OCaml上的那一部分，你必须掌握了BuckleScript基础知识之后才能更好的阅读开始后面的章节

Reason

Reason是Ocaml另一种语法实现，对JS程序员更友好，能降低Ocaml的学习成本 Reason运行在Ocaml之上，尽管语法不同，但其他的东西基本都是相同的 这也意味着Reason也可以运行在BuckleScript上 BuckleScript也对一些Reason功能进行了优先的支持

OPAM

OPAM是Ocaml的官方包管理工具，而BuckleScript使用NPM和Yarn，作为初学者你不必了解OPAM，在后面有些高级应用中将会提到它。

External/Interop/FFI

这些都是BuckcleScript与Javascript转换会用到的一些术语 <-> 无

External：在BuckleScript中，External是使用JS值的主要方式。

Interop：interoperability 互用性的缩写。

FFI：Foreign Function Interface , 外部函数接口，是对像external，wrapper 和 interop的统一称呼，基本意思就是调用其他语言。
