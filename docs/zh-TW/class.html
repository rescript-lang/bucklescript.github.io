new

使用bs.new 來仿造，例如：new Date()

type t
external createDate : unit -> t = "Date" [@@bs.new]

let date = createDate ()

type t;
[@bs.new] external createDate : unit => t = "Date";

let date = createDate();

輸出

var date = new Date();

如果你引入的 JS 模組 (module) 本身是類別 (class) ，你可以串接 bs.new 和 bs.module

type t
external book : unit -> t = "Book" [@@bs.new] [@@bs.module]
let bookInstance = book ()

type t;
[@bs.new] [@bs.module] external book : unit => t = "Book";
let bookInstance = book();

輸出

var Book = require("Book");
var bookInstance = new Book();

綁定到 JS 類別 (class)

JS 類別實際上只是 JS 物件串接起來。一般情況下，我們會傾向於使用前面 object 的章節裡的方法來綁定一個 JS 物件。

OCaml having classes really helps with modeling JS classes. Just add a [@bs] to a class type to turn them into a Js.t class:

class type _rect = object
  method height : int [@@bs.set]
  method width : int [@@bs.set]
  method draw : unit -> unit
end [@bs]

type rect = _rect Js.t

class type _rect =
  [@bs]
  {
    [@bs.set] pub height: int;
    [@bs.set] pub width: int;
    pub draw: unit => unit
  };

type rect = Js.t(_rect);

對於 Js.t 的類別，使用箭頭型別的函式會被視為真的函式（自動被標記為 [@bs.meth] ），而沒有使用箭頭型別的函式則被視為屬性 (properties) 。 新增 bs.set 到那些函式會讓他們可變 (mutable) ，如此一來你就可以使用 #= 來改變它們。 移除 bs.set 的話，會讓此函式/屬性不可變 (immutable) 。 基本上就如同 object 章節裡的特性一樣。
